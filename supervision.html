====Lancement d'une VM avec deux interfaces réseaux==== Il faut pour cela répéter le couple -net nic MAC avec les bon paramètres réseau et bien faire gaffe à pas mettre les mêmes addresses MAC. 
                        kvm -m 2G -hda alimroivili-Routeur.qcow2 -net nic,macaddr=42:70:10:01:01:01 -net vde,sock=/var/run/vde2/kvmstud7002.ctl -net nic,macaddr=42:70:10:01:01:02 -net vde,sock=/var/run/vde2/kvmstudPriv7002.ctl -vnc :02 -nographic
====Lancement d'une VM avec deux interfaces réseaux==== 
vinagre thorin:02
====Affectation des adresses IP==== Afin d'affecter des adresses statiques à nos interfaces, il faut éditer le fichier /etc/network/interfaces en adéquation avec les adresses indiquées sur mon dessin. 
                        allow-hotplug ens3
                        iface ens3 inet static
                        address 10.244.2.1/24
                        gateway 10.4.2.10

                        allow-hotplug ens4
                        iface ens4 inet static
                        address 10.4.2.1/24
                        gateway 10.4.2.10
*'''activer le routage :''' Crée un fichier routeur.conf dans /etc/sysctl.d/
Dans se fichier j'écris : "net.ipv4.ip_forward=1"
Ensuite : sysctl -p /etc/sysctl.d/routeur.conf
Et normalement la valeur du fichier /proc/sys/net/ipv4/ip_forward est passer de 0 à 1 ce qui signifie que le routage est maintenant activé. ====Utilisation tshark==== Il faut installer le paquet tshark sur le routeur, et pour ça il faut changer l'ip du dns en l'ip 10.4.10.1.

Et maintenant je peux installer le paquet tshark : 
apt install tshark
Et je peux l'utiliser donc je le lance sur l'interface ens3 : 
tshark -i ens3

====NFS====
====Configuration du serveur :====
Installation NFS :
sudo apt install nfs-kernel-server
Crée le répertoire :
mkdir NFS_serveur
Chemin de NFS_serveur :
 /home/alimroivili/NFS_serveur
Exportation du répertoire : 
 sudo nano /etc/exports
Ecrire: 
/home/alimroivili/NFS_serveur 10.144.46.33(rw,sync)
Redémarrage du service NFS : 
 sudo service nfs-kernel-server restart

====Configuration du client:====
Installation du paquet NFS : 
 sudo apt install nfs-common
Visualisation de la liste d'exportation : 
 showmount -e 10.144.46.1
l'exportation de NFS_serveur est disponible pour le client. Création d'un répertoire : 
 mkdir NFS_client
refaire les étapes suivantes à chaque démarrage du client pour les resynchroniser : Passage en root :
 su -
Synchronisation entre NFS_serveur et NFS_client : 
mount 10.144.46.1:/home/alimroivili/NFS_serveur /home/alimroivili/NFS_client
Vérification du montage : 
 df -h /home/alimroivili/NFS_client

====SAMBA====
====Configuration du serveur :====
Installation SAMBA : 
sudo apt install samba
Crée le répertoire : 
mkdir Samba_serveur
Chemin de Samba_serveur : 
 /home/alimroivili/Samba_serveur
Édition du fichier de configuration Samba : 
                        sudo nano /etc/samba/smb.conf
--> Ecrire : 
[samba]
                            path = /home/alimroivili/Samba_serveur
                            browseable = yes
                            writeable = yes
                            valid users = alimroivili
Ajout d'un utilisateur avec son mot de passe : 
 sudo smbpasswd -a alimroivili
Redémarrage du service : 
 sudo systemctl restart smbd

====Configuration du client:====
Installation du paquet Samba : 
sudo apt install cifs-utils
Création d'un répertoire : 
 mkdir Samba_client
Chemin de Samba_client : 
 /home/alimroivili/Samba_client
Synchronisation entre Samba_serveur et Samba_client : 
 sudo mount -t cifs //10.144.46.1/samba /home/alimroivili/Samba_client -o username=alimroivili
Vérification du montage : 
 df -h /home/alimroivili/Samba_client
